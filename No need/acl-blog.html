Access Control Lists (ACLs)

Securing the Network with Traffic Filtering

Access Control Lists (ACLs) are a set of rules used by routers to filter network traffic based on specific criteria. They act as a basic firewall, allowing or denying packets to control access to network resources and improve security.

How ACLs Work

  • Sequential Processing: Rules are checked from top to bottom. Once a match is found, the action (permit/deny) is taken, and no further rules are checked.
  • Implicit Deny: At the very end of every ACL, there is an invisible "deny all" rule. If a packet doesn't match any explicit rule, it is dropped.
  • Directional: ACLs must be applied to an interface in a specific direction: Inbound (filtering traffic entering the router) or Outbound (filtering traffic leaving the router).

Standard vs. Extended ACLs

Feature Standard ACL Extended ACL
Filtering Criteria Source IP address only. Source/Dest IP, Protocol (TCP/UDP), and Port numbers.
Number Range 1-99 and 1300-1999. 100-199 and 2000-2699.
Best Practice Placement Closest to the Destination. Closest to the Source.

Wildcard Masks

ACLs use Wildcard Masks (the inverse of a subnet mask) to determine which bits of an IP address to examine. For example, a 0.0.0.255 wildcard mask tells the router to ignore the last octet, matching an entire Class C subnet.

Basic Configuration Syntax

Router> enable
Router# configure terminal
! Create a Standard ACL to permit a specific subnet
Router(config)# access-list 10 permit 192.168.1.0 0.0.0.255
! Apply the ACL to an interface
Router(config)# interface gigabitEthernet 0/0
Router(config-if)# ip access-group 10 out
