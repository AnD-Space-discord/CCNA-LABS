OSPF Single Area Lab

Dynamic Routing Configuration Using OSPFv2

Objective

Configure Open Shortest Path First (OSPF) in a single area (Area 0) to establish full connectivity between three routers. This lab covers basic router hardening, interface addressing, and OSPF process configuration.

Lab Topology

Topology: R1 ? R2 ? R3 connected via point-to-point serial/ethernet links.

Device Interface IP Address Subnet Mask Description
R1 GigabitEthernet0/0 10.1.1.1 255.255.255.0 LAN ľ R1 Users
R1 Serial0/0/0 192.168.12.1 255.255.255.252 Link to R2
R2 GigabitEthernet0/0 10.2.2.1 255.255.255.0 LAN ľ R2 Users
R2 Serial0/0/0 192.168.12.2 255.255.255.252 Link to R1
R2 Serial0/0/1 192.168.23.2 255.255.255.252 Link to R3
R3 GigabitEthernet0/0 10.3.3.1 255.255.255.0 LAN ľ R3 Users
R3 Serial0/0/1 192.168.23.3 255.255.255.252 Link to R2

This command applies a type-7 cipher to all plaintext passwords stored in the running config. While not cryptographically strong, it prevents shoulder-surfing when the config is displayed on screen. enable secret uses MD5 (type-5) and is always preferred over enable password.

Task 2 ľ Interface Configuration (All Routers)

Assign IP addresses and descriptions to each interface, then bring it up with no shutdown.

R1 ľ Interface Config

R1(config)# interface GigabitEthernet0/0
R1(config-if)# description LAN ľ R1 Users
R1(config-if)# ip address 10.1.1.1 255.255.255.0
R1(config-if)# no shutdown
R1(config-if)# exit
R1(config)# interface Serial0/0/0
R1(config-if)# description Link to R2
R1(config-if)# ip address 192.168.12.1 255.255.255.252
R1(config-if)# clock rate 128000
R1(config-if)# no shutdown
R1(config-if)# exit

R2 ľ Interface Config

R2(config)# interface GigabitEthernet0/0
R2(config-if)# description LAN ľ R2 Users
R2(config-if)# ip address 10.2.2.1 255.255.255.0
R2(config-if)# no shutdown
R2(config-if)# exit
R2(config)# interface Serial0/0/0
R2(config-if)# description Link to R1
R2(config-if)# ip address 192.168.12.2 255.255.255.252
R2(config-if)# no shutdown
R2(config-if)# exit
R2(config)# interface Serial0/0/1
R2(config-if)# description Link to R3
R2(config-if)# ip address 192.168.23.2 255.255.255.252
R2(config-if)# clock rate 128000
R2(config-if)# no shutdown
R2(config-if)# exit

R3 ľ Interface Config

R3(config)# interface GigabitEthernet0/0
R3(config-if)# description LAN ľ R3 Users
R3(config-if)# ip address 10.3.3.1 255.255.255.0
R3(config-if)# no shutdown
R3(config-if)# exit
R3(config)# interface Serial0/0/1
R3(config-if)# description Link to R2
R3(config-if)# ip address 192.168.23.3 255.255.255.252
R3(config-if)# no shutdown

clock rate on DCE interfaces In Packet Tracer, the serial interface acting as the DCE (typically the one with the cable's clock symbol) needs clock rate set. Use show controllers Serial0/0/0 to confirm which end is DCE before applying.
